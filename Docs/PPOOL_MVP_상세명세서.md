## 1. 시스템 개요

### 1-1. 기술 스택 (권장)
| 영역 | 기술 |
|---|---|
| 프론트엔드 | React (PWA 지원) |
| 백엔드·DB | Supabase (PostgreSQL + Auth + Realtime) |
| 인증 | 카카오 소셜 로그인 (Supabase Auth 연동) |
| 지도 | 카카오맵 API |
| 배포 | Vercel |
| 알림 | 웹 푸시 (Service Worker) |

### 1-2. 사용자 유형
- **일반 회원**: 운전자·탑승자 역할을 모두 가질 수 있음
- **관리자**: 봉사부장, 총무 (승인/삭제/백업 권한)

## 3. 화면별 상세 명세

### 3-1. [S01] 로그인 페이지 `/login`

**구성 요소**
- P플 로고 (중앙 상단, 엄지손가락 아이콘 + 텍스트)
- 서브 카피: "지파 청년부 카풀 서비스"
- **카카오 로그인 버튼** (노란색 공식 가이드 준수)
- 약관 안내 문구: "로그인 시 이용약관 및 개인정보 처리방침에 동의한 것으로 간주됩니다."
- [이용약관] [개인정보 처리방침] [위치정보 이용약관] 링크

**동작**
1. 카카오 로그인 버튼 클릭 → 카카오 OAuth 진행
2. 최초 로그인이면 → [S02] 회원가입 페이지로 이동
3. 가입 완료 상태 + 승인 대기 → [S03] 승인 대기 페이지
4. 가입 완료 상태 + 승인 → [S04] 메인 페이지
5. BANNED 상태 → 이용 정지 안내 페이지

---

### 3-2. [S02] 회원가입 정보 입력 페이지 `/signup`

**구성 요소**
- 안내 문구: "청년부 구성원 확인을 위해 아래 정보를 입력해주세요."
- 입력 필드
  - 이름 (필수)
  - 부서 (드롭다운, 필수) — 지파 부서 목록 관리자가 사전 등록
  - 휴대폰 번호 (필수, 형식 검증: 010-XXXX-XXXX)
  - 성별 (선택, 라디오: 남/여/미지정)
- 약관 동의 체크박스 3개
  - (필수) 이용약관
  - (필수) 개인정보 처리방침
  - (선택) 위치정보 이용약관
- 전체 동의 체크박스
- [가입 신청] 버튼

**동작**
- 필수 항목 미입력/미동의 시 버튼 비활성화
- 제출 시 → DB에 status='PENDING'으로 저장 → [S03] 승인 대기 페이지

---

### 3-3. [S03] 승인 대기 페이지 `/pending`

**구성 요소**
- 아이콘 + "관리자 승인 대기 중입니다"
- 안내: "청년부 구성원 확인 후 승인됩니다. 통상 1~2일 소요됩니다. 문의는 카카오톡 채널로 주세요."
- [카카오톡 채널 문의] 버튼
- [로그아웃] 버튼

### 3-4. [S04] 메인 페이지 `/` (홈)

**구성 요소**

```
┌─────────────────────────────────┐
│  P플 로고    🔔 알림   👤 MY   │  ← 상단 헤더
├─────────────────────────────────┤
│  [🚗 운전해요]  [🙋 태워주세요] │  ← CTA 2개 (큰 버튼)
├─────────────────────────────────┤
│  🔍 [지역 검색]                 │  ← 필터
│  [전체] [동성만] [잔여석]       │  ← 필터 토글
├─────────────────────────────────┤
│ ┌─────────────────────────────┐ │
│ │ [운전자] 광주 → 서울         │ │
│ │ 03/31 14:00 · 3/4명 · 3,000원│ │  ← 카드 리스트
│ │ 홍길동 (청년1부) · 남        │ │
│ └─────────────────────────────┘ │
│ ┌─────────────────────────────┐ │
│ │ ...                          │ │
│ └─────────────────────────────┘ │
└─────────────────────────────────┘
```

**기능**
- 실시간 게시글 리스트 (최신순 또는 출발 임박순)
- 필터: 지역, 성별 매칭, 잔여석 유무
- 무한 스크롤 또는 페이지네이션
- 카드 클릭 시 → [S07] 게시글 상세
- "운전해요" 클릭 → [S05] 운전자 게시글 작성
- "태워주세요" 클릭 → [S06] 탑승자 신청 (또는 게시글 찾기 안내)

**카드 정보 표시**
- 역할 뱃지 (운전자=주황 / 탑승자=파랑)
- 출발지 → 도착지
- 출발 시간
- 현재 인원 / 최대 인원
- 분담금 (항상 3,000원 고정 표시)
- 게시자 이름, 부서
- 성별 아이콘

### 3-5. [S05] 운전자 게시글 작성 페이지 `/rides/new`

**구성 요소**

| 필드 | 형식 | 필수 | 비고 |
|---|---|---|---|
| 출발지 | 카카오맵 주소 검색 | ✅ | 현재 위치 버튼 |
| 목적지 | 카카오맵 주소 검색 | ✅ | |
| 경유지 | 주소 검색 (최대 3개) | | + 추가 버튼 |
| 출발 시각 | 날짜+시간 피커 | ✅ | 현재 시각 +10분 이상 |
| 모집 인원 | 숫자 (1~차량 최대) | ✅ | |
| 차량 선택 | 등록된 차량 목록 | ✅ | 미등록 시 등록 유도 |
| 성별 매칭 | 전체 / 동성만 | | 기본값: 전체 |
| 공지사항 | 텍스트 (100자) | | "짐 많으시면 미리 알려주세요" 등 |
| 분담금 | **3,000원 고정 (수정 불가)** | | |

**하단 버튼**
- [취소] [등록하기]

**유의사항 표시**
- "출발 5분 내에 '출발' 버튼을 누르지 않으면 게시글이 자동 삭제됩니다."
- "등록 시 이용약관의 운전자 의무에 동의한 것으로 간주됩니다."

### 3-7. [S07] 게시글 상세 페이지 `/rides/[id]`

**구성 요소**

```
┌─────────────────────────────────┐
│  ← 뒤로                         │
├─────────────────────────────────┤
│  [운전자] 모집 중 (3/4)         │
│                                 │
│  광주 → 서울                    │
│  (카카오맵 경로 표시)           │
│                                 │
│  ⏰ 03/31 14:00 출발            │
│  💰 1인당 3,000원 (유류비 분담) │
│  🚗 쏘렌토 / 12가 3456          │
│                                 │
│  운전자: 홍길동 (청년1부) · 남  │
│  📢 짐 많으시면 미리 알려주세요 │
├─────────────────────────────────┤
│  탑승 희망 위치: [주소 검색]    │
│  [신청하기]                     │
└─────────────────────────────────┘
```

**운전자가 본인 게시글 접근 시**
- [출발] 버튼 (출발 시각 ±5분 내 활성화)
- [수정] [삭제] 버튼
- 신청자 목록 (승인/거부 버튼)

**탑승자가 접근 시**
- 잔여석이 있으면 [신청하기] 버튼
- 이미 신청한 경우 [신청 취소] 버튼
- 승인된 경우 [운전자에게 연락] 버튼 (전화걸기)

**상태별 표시**
- 모집 중: 녹색 뱃지
- 마감: 회색 뱃지
- 출발함: 주황 뱃지 (실시간 위치 표시, 동의한 경우)
- 완료: 회색 뱃지

### 3-8. [S08] 마이페이지 `/my`

**섹션 구성**
1. **내 프로필**
   - 이름, 부서, 연락처, 성별
   - [프로필 수정] 버튼

2. **운전자 정보** (운전자 등록 시)
   - 차량번호, 차종, 최대 탑승 인원
   - [차량 정보 수정] 버튼
   - 미등록 시: [운전자 등록하기] 버튼

3. **계좌 정보**
   - 은행, 계좌번호 (뒷자리 마스킹)
   - [수정] 버튼

4. **내 활동**
   - 내가 올린 운전자 게시글 (진행 중 / 완료)
   - 내 탑승 신청 내역 (대기 / 승인 / 거절 / 완료)

5. **설정**
   - 알림 설정
   - 이용약관 / 개인정보 처리방침 보기
   - [로그아웃] [회원탈퇴]

### 3-10. [S10] 관리자 페이지 `/admin`

**접근 권한**: 관리자 계정만 접근 가능

**메뉴 구성**

#### 탭 1: 회원 관리
- 승인 대기 목록 (이름, 부서, 연락처, 가입일)
  - [승인] [거부] 버튼
- 전체 회원 목록 (검색, 필터)
  - 상태별 필터 (승인/정지/탈퇴)
  - 노쇼·신고 누적 회원 강조 표시
  - [정지] [해제] 버튼

#### 탭 2: 신고 관리
- 대기 중인 신고 목록
- 신고 내용 확인 → [처리 완료] [반려] 버튼
- 처리 시 피신고자 제재 결정

#### 탭 3: 게시글 관리
- 활성 게시글 목록
- 부적절 게시글 [삭제] 버튼

#### 탭 4: 백업
- [전체 회원 엑셀 다운로드]
- [게시글 이력 엑셀 다운로드] (기간 선택)
- [신고 이력 엑셀 다운로드]
- 마지막 백업일시 표시

#### 탭 5: 통계 (선택)
- 일별 가입자 수
- 일별 게시글 수
- 매칭 성공률

## 4. API 엔드포인트

> Supabase 사용 시 대부분 자동 생성되지만, 권한 정책(RLS)은 수동 설정 필요.

## 5. 사용자 플로우

### 5-1. 신규 가입 플로우
```
로그인 페이지
  → 카카오 로그인
  → [최초] 회원가입 정보 입력
  → 약관 동의 체크
  → 가입 신청 완료
  → 승인 대기 페이지
  → (관리자 승인)
  → 메인 페이지
```

### 5-2. 운전자 카풀 등록 플로우
```
메인 페이지
  → [운전해요] 클릭
  → (최초) 운전자 등록 (차량 정보)
  → 게시글 작성 페이지
  → 출발지/목적지/시간/인원 입력
  → 등록하기
  → 게시글 상세 페이지 (본인 게시글 뷰)
  → 신청자 대기
  → 승인/거부
  → 출발 시각 근처 → [출발] 버튼 클릭
  → 운행 → [완료] 버튼 클릭
  → 분담금 정산 (당사자 직접)
```

### 5-3. 탑승자 신청 플로우
```
메인 페이지
  → 게시글 리스트 스크롤
  → 원하는 게시글 클릭
  → 게시글 상세
  → 탑승 희망 위치 입력
  → [신청하기]
  → 대기 상태
  → (운전자 승인)
  → 승인 알림
  → 출발 시간 맞춰 탑승
  → 완료 후 3,000원 직접 정산
```

### 5-4. 신고 플로우
```
게시글 상세 또는 마이페이지 활동 내역
  → [신고] 버튼
  → 신고 사유 선택 (노쇼/무례/위험운전/요금문제/기타)
  → 상세 내용 입력 (선택)
  → 제출
  → 관리자 검토
  → 결과 알림
```

## 6. 상태값 정의

### 6-1. 회원 상태 (users.status)
| 값 | 설명 | 기능 제한 |
|---|---|---|
| PENDING | 가입 신청, 승인 대기 | 로그인만 가능, 서비스 이용 불가 |
| APPROVED | 승인 완료 | 전체 이용 가능 |
| REJECTED | 가입 거부 | 재가입 신청 가능 |
| BANNED | 정지 | banned_until까지 이용 불가 |

### 6-2. 게시글 상태 (rides.status)
| 값 | 설명 | 전환 조건 |
|---|---|---|
| OPEN | 모집 중 | 생성 시 기본값 |
| FULL | 정원 마감 | current_seats == max_seats |
| STARTED | 출발함 | 운전자가 [출발] 버튼 클릭 |
| COMPLETED | 완료 | 운전자가 [완료] 버튼 클릭 |
| CANCELLED | 취소 | 운전자가 삭제 or 5분 내 출발 안 누름 |

### 6-3. 신청 상태 (ride_requests.status)
| 값 | 설명 |
|---|---|
| PENDING | 신청, 운전자 확인 대기 |
| APPROVED | 운전자가 승인 |
| REJECTED | 운전자가 거부 |
| CANCELLED | 탑승자가 취소 |
| NO_SHOW | 노쇼 처리 |
| COMPLETED | 운행 완료 |

## 7. 에러 처리 및 예외

### 7-1. 주요 엣지 케이스

| 상황 | 처리 |
|---|---|
| 출발 5분 내 [출발] 미클릭 | 게시글 자동 CANCELLED, 신청자에게 알림 |
| 운전자가 게시글 삭제 | 승인된 신청자들에게 즉시 알림 |
| 탑승자가 신청 후 승인 전 취소 | 기록만 남기고 정상 삭제 |
| 탑승자가 승인 후 출발 30분 이내 취소 | 취소 기록 +1 |
| 탑승자가 출발 10분 이내 취소 | no_show_count +1 |
| 모집 인원 초과 신청 | FIFO 처리, 운전자가 선택 |
| 동시 신청으로 정원 초과 | 트랜잭션으로 원자적 처리 |
| 정지된 회원의 API 호출 | 401 반환, 로그아웃 강제 |

### 7-2. 유효성 검증

- 출발 시각: 현재 시각 +10분 이상, +7일 이내
- 모집 인원: 1 이상 차량 max_passengers 이하
- 연락처: 정규식 검증 `^01[0-9]-?[0-9]{3,4}-?[0-9]{4}$`
- 차량번호: 한국 차량번호 패턴 검증
- 텍스트 필드: XSS 방지 sanitize

## 8. 보안 요구사항

### 8-1. 인증·인가
- [ ] 모든 API는 JWT 토큰 검증
- [ ] 관리자 API는 role === 'admin' 확인
- [ ] Supabase RLS(Row Level Security) 정책 설정
  - 내 게시글만 수정·삭제 가능
  - 내 신청 내역만 조회 가능
  - 관리자는 전체 접근

### 8-2. 데이터 암호화
- [ ] 계좌번호: **AES-256 암호화** 후 DB 저장
- [ ] 전화번호: 해시 또는 부분 암호화
- [ ] HTTPS 필수 (Vercel 기본 제공)

### 8-3. 개인정보 노출 방지
- [ ] 타 회원에게는 **이름, 부서, 성별**만 공개
- [ ] 연락처는 **신청 승인 후**에만 공개
- [ ] 계좌번호는 **본인에게만** 공개 (운전자 → 본인 마이페이지)
- [ ] 화면 스크린샷에 워터마크 (유저 ID 해시값)

### 8-4. 로그 관리
- [ ] 승인/거부, 제재, 신고 처리 등 관리자 액션 로그
- [ ] 로그인 기록 3개월 보관
- [ ] 개인정보 접근 로그

### 8-5. 기타
- [ ] CSRF, XSS, SQL Injection 방어 (프레임워크 기본 + 수동 검증)
- [ ] Rate Limiting (동일 IP 분당 요청 제한)
- [ ] 비정상 트래픽 탐지 알림
