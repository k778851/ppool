# P플 카풀 웹 프로젝트 — 기획 보완 문서

> **프로젝트 개요**
> - **대상**: 지파 청년부 구성원
> - **출시 형태**: 웹 우선 (이후 필요 시 앱 확장)
> - **요금 정책**: **인당 2,000~3,000원 정액** (유류비 분담 성격)
> - **관리자**: 봉사부장(정) / 총무(부)

---

## 1. 법적 이슈 및 대응

### 1-1. 유상운송 금지 조항 대응

한국 여객자동차운수사업법상 자가용 유상운송은 원칙 금지이며, 정해진 요금을 받으면 문제가 될 수 있습니다. **인당 2~3천원 정액** 구조를 유지하되, 아래 조치가 반드시 필요합니다.

- [ ] 서비스를 **"유류비 분담"** 개념으로 명확히 정의 (이용약관·UI 문구 반영)
- [ ] 금액을 **"요금"이 아닌 "분담금"**으로 표기
- [ ] "본 서비스는 영리 목적이 아니며, 지파 청년부 구성원 간 이동 편의를 위한 비영리 내부 서비스"임을 약관 1조에 명시
- [ ] 비구성원(외부인) 이용 원천 차단 (로그인 승인 단계에서 거름)
- [ ] 운전자에게 "이익 목적의 영업 운행 금지" 동의 받기

> 💡 쿠팡버스·풀러스 사례처럼 **출퇴근 목적 카풀**은 법적으로 인정됩니다. 교회 예배·모임 이동이 이에 준하는 "정기 이동" 성격이라는 점을 약관에서 뒷받침하는 것이 안전합니다.

### 1-2. 이용약관 / 개인정보 처리방침 초안 필요

- [ ] **이용약관** (서비스 성격, 금지행위, 책임 한계, 탈퇴)
- [ ] **개인정보 처리방침** (수집 항목, 보관기간, 파기, 제3자 제공 여부)
- [ ] **위치정보 이용약관** (위치공유 기능이 있으므로 별도 필요)
- [ ] 회원가입 시 **3종 체크박스** 동의 필수

## 2. 보험 및 사고 책임

### 2-1. 문제 인식
개인 자동차보험은 유상운송 중 사고 시 **면책되는 경우**가 많음. 지파가 플랫폼 운영자이므로 분쟁 시 연관 책임 논란 가능.

### 2-2. 대응 방안
- [ ] 약관에 **면책 조항** 명시: "본 서비스는 매칭 플랫폼이며, 운행 중 사고·분쟁에 대한 모든 책임은 당사자에게 있음"
- [ ] 운전자 등록 시 **자동차보험 가입 여부 확인** (필수 체크)
- [ ] 사고 발생 시 **플랫폼 책임 없음** 동의 받기
- [ ] 장기적으로 지파 차원의 **단체보험** 검토 (청년부 공식 활동 연장선)

## 3. 개인정보 보호

### 3-1. 수집 항목 재검토

| 항목 | 현재 기획 | 권장 |
|---|---|---|
| 이름 | 수집 | 유지 (닉네임 병행 허용) |
| 부서 | 수집 | 유지 |
| 연락처 | 수집 | 유지 (필수) |
| 성별 | 필수 수집 | **선택** 또는 "여성 안심 매칭" 옵션으로만 |
| 계좌번호 | 수집 | **암호화 저장** 필수, 가능하면 **카카오페이 송금 링크**로 대체 |
| 차량번호 | 수집 | 유지 (운전자만) |
| 위치정보 | 실시간 공유 | 운행 중에만, 하차 후 자동 삭제 |

### 3-2. 워터마크 재검토
전화번호 워터마크는 좋은 발상이나 **전화번호 자체가 개인정보**이므로 유출 경로가 될 수 있음.
- [ ] **유저 ID 해시값** 또는 **랜덤 닉네임 + 가입일시**로 대체 권장
- [ ] 스크린샷 유출 시 추적 가능한 정도면 충분

### 3-3. 계좌번호 처리
- [ ] DB에 평문 저장 금지 → **AES 암호화** 적용
- [ ] 이상적으로는 **카카오페이 송금 링크** 자동 생성 방식으로 계좌번호 노출 자체를 줄이기

## 4. MVP 범위 확정 (웹 우선)

### 4-1. 1차 출시 (MVP) — 필수 기능만
- [x] 로그인 (카카오 소셜 + 관리자 승인 대기)
- [x] 메인 피드 (운전자 게시글 리스트)
- [x] 운전자 게시글 등록/수정/삭제
- [x] 탑승자 신청/취소
- [x] 관리자 페이지 (승인, 명단, 신고 처리, 백업)
- [x] 문의 → **카카오톡 채널** 연결 (직접 개발 X)

### 4-2. 2차 확장 (MVP 안정화 후)
- 택시 공유 기능
- 평점/신고 시스템
- 재난 문자 스타일 알림
- 오픈채팅 연결
- 출발·도착 알림 푸시

### 4-3. 3차 (사용자 데이터 확보 후 판단)
- 피드형 커뮤니티 기능
- 앱 전환 (iOS 포함)

> ⚠️ MVP에 모든 걸 넣으면 **출시 자체가 지연**됩니다. "없어도 운영 가능한가?" 기준으로 2·3차로 미루세요.

## 5. 기능별 보완 사항

### 5-1. 요금 정책 세부화
- [ ] 기본 분담금: **2,000~3,000원** (거리 무관 정액? 또는 단거리/장거리 2단 구분?)
- [ ] 지역장/봉사부장과 협의하여 **단일 가이드** 확정
- [ ] 운전자가 임의로 금액 올리는 것 방지 (시스템 고정값)

### 5-2. 노쇼(No-show) 대응
- [ ] 탑승자 노쇼 시 **운전자가 신고 버튼** 누르면 기록
- [ ] 누적 3회 시 일정 기간 이용 제한
- [ ] 운전자 노쇼(출발 안 함)도 동일하게 관리

### 5-3. 취소 정책
| 시점 | 정책 |
|---|---|
| 출발 30분 전 이상 | 자유 취소 |
| 출발 10~30분 전 | 취소 기록 남김 |
| 출발 10분 이내 | 노쇼 처리 |

### 5-4. 분쟁 처리
- [ ] **관리자 중재 채널** 마련 (카카오톡 채널로 통합)
- [ ] 게시글/채팅 **기록 최소 30일 보존** (분쟁 대비)
- [ ] 반복 신고자 조치 기준 문서화

### 5-5. 평점 시스템 (2차)
- [ ] 간단한 👍/🚨(신고) 버튼만
- [ ] 별점은 운영 부담 크므로 보류

### 5-6. 긴급 상황 대응 (2차)
- [ ] 112/119 **원터치 신고 버튼**
- [ ] 지인에게 **위치공유 링크 전송** 기능

### 5-7. 여성 안심 기능
- [ ] 성별 기반 **동성 우선 매칭** 옵션
- [ ] 여성 운전자 전용 게시글 필터

## 6. 로그인 방식 — 최종안

**카카오 소셜 로그인 + 관리자 수동 승인** 하이브리드 권장.

### 이유
- 비밀번호 관리 리스크 없음 (3안 탈락 사유)
- 위아원/원패스는 총회 승인 2개월+ 필요 → 출시 지연 (1안 탈락 사유)
- 관리자 승인으로 **비구성원 차단** → 법적 "내부 서비스" 근거 확보

### 플로우
1. 사용자가 카카오로 로그인
2. 부서/이름/연락처 추가 입력
3. **"승인 대기" 상태**로 DB 저장 (기능 이용 불가)
4. 관리자가 명단 확인 후 승인
5. 승인 완료 후 정상 이용

## 7. 기술 고려사항

### 7-1. 비용 예산 (월 기준 예상)
| 항목 | 예상 비용 | 비고 |
|---|---|---|
| 서버 (AWS/Vercel 등) | 1~5만원 | 트래픽 따라 |
| DB (Supabase/Firebase) | 0~3만원 | 프리티어 활용 가능 |
| 카카오 지도 API | 무료~ | 월 30만 건까지 무료 |
| 도메인 | 연 1~2만원 | .com 기준 |
| **월 총합 예상** | **3~10만원** | |

- [ ] 예산 주체 확정 필요 (지파 예산 / 청년회비 / 후원)
- [ ] iOS 앱 확장 시 **개발자 계정 연 $99 + 심사 2~4주** 추가

### 7-2. 기술 스택 제안
- **프론트**: React 또는 Next.js (PWA 적용 쉬움)
- **백엔드**: Supabase 또는 Firebase (초기 비용 절감)
- **지도**: 카카오맵 API
- **알림**: PWA 푸시 (웹에서도 가능, iOS 제약 있음)

### 7-3. PWA 권장
웹에서 시작하더라도 **PWA(Progressive Web App)** 로 만들면:
- 홈 화면에 아이콘 추가 가능 (앱처럼 보임)
- 푸시 알림 일부 지원
- 오프라인 기본 동작
- 앱스토어 심사 불필요

## 9. 디자인 가이드

### 9-1. 색상 규칙 (혼동 방지)
| 용도 | 색상 |
|---|---|
| 메인 브랜드 | 민트 |
| 운전자 게시글/버튼 | 주황 |
| 탑승자 게시글/버튼 | 파랑 |
| 경고/신고 | 빨강 (추가) |

### 9-2. 화면 흐름 (MVP)
1. **로그인 페이지** — 카카오 로그인 버튼 중심
2. **메인 페이지** — 운전자 게시글 피드 + 상단 CTA(구해요/모아요)
3. **게시글 작성 페이지** — 운전자/탑승자 구분
4. **게시글 상세 페이지** — 신청·취소 버튼
5. **마이페이지** — 내 게시글/신청 내역
6. **관리자 페이지** — 별도 경로(`/admin`)

### 9-3. UX 유의사항
- [ ] 영상 자동재생은 **음소거 필수** (브라우저 정책)
- [ ] 영상 on/off 토글 제공 (데이터·배터리 고려)
- [ ] 알림창은 "재난 문자" 스타일은 임팩트 있으나 **과도한 사용 자제** (피로도)
- [ ] 접근성: 글자 크기 조절, 색맹 대응(색만으로 정보 전달 X)
